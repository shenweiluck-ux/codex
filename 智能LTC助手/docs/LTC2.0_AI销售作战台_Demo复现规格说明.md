# LTC 2.0 AI 销售作战台 Demo 复现规格说明

本文档用于让另一个 AI 或开发者基于当前项目对话，复现一个与现有 Demo 功能和界面体验基本一致的原型系统。

目标不是复现某个传统 CRM 页面，而是复现一个“以销售为中心、由实时客户信息驱动、能够动态调整商机推进路径和组织资源支持”的 AI 销售作战系统。

## 1. 产品定位

产品名称：

**LTC 2.0 AI 销售作战台**

一句话定位：

> 销售输入真实客户现场信息，系统实时理解业务信号，动态更新商机状态、推进路径、风险、任务、资源建议和商机记忆。

核心理念：

- 不是“让销售填流程”，而是“让销售和一个懂业务的 AI 作战助手一起打单”。
- 不是先固化 LTC 阶段，而是根据客户现场信息动态调整推进路径。
- 销售输入的信息越真实、越完整，系统越能帮助销售争取公司资源、缩短审批流程、提高赢单概率。

## 2. 当前 Demo 技术形态

当前 Demo 是一个纯前端静态原型：

- `frontend/index.html`
- `frontend/styles.css`
- `frontend/app.js`

数据保存在浏览器 `localStorage` 中。

当前没有真实后端、数据库、登录、AI API。  
AI 能力由前端规则引擎模拟，包括关键词识别、状态评分、风险触发、动态路径插入/移除和相关项目召回。

项目结构建议：

```text
project-root/
  frontend/
    index.html
    styles.css
    app.js
    manifest.json
  backend/
    server.js
  database/
    schema.sql
  docs/
    LTC2.0_AI销售作战台_Demo复现规格说明.md
```

## 3. 页面结构

系统有 4 个主栏目：

1. AI 销售作战台
2. 商机记忆
3. 资源加速
4. 管理驾驶舱

导航在 PC 端位于左侧深色侧栏，在移动端变成底部固定导航。

### 3.1 AI 销售作战台

这是系统主界面。

布局采用三块结构：

```text
┌──────────────────────────────────────────┐
│ 上方：AI 输入与业务理解区                 │
│ - AI Hero                                │
│ - 实时输入框                              │
│ - 实时关键词高亮                          │
│ - AI 理解卡                               │
└──────────────────────────────────────────┘

┌──────────────────────┬───────────────────┐
│ 左下：商机队列与任务  │ 右下：商机实时画像 │
│ - 商机卡片            │ - 赢率环形图       │
│ - 下一步动作          │ - AI 诊断          │
│                       │ - 动态推进路径     │
│                       │ - 识别信号         │
│                       │ - 风险预警         │
└──────────────────────┴───────────────────┘
```

上方区域占主要空间，因为它是销售的核心工作区。  
销售可以录入客户现场信息，也可以借助“插入测试样例”快速演示。

### 3.2 商机记忆

商机记忆是独立栏目，不依赖作战台当前选中的商机。

布局为左中右三栏：

```text
┌──────────────┬────────────────────────┬────────────────────┐
│ 左：商机索引  │ 中：记忆流水             │ 右：关键信息面板     │
│ - 全部商机    │ - 全部商机流水           │ - 关键人             │
│ - 单个商机    │ - 或单商机流水           │ - 相关项目与资料     │
└──────────────┴────────────────────────┴────────────────────┘
```

左侧：

- `全部商机`
- 每个具体商机

中间：

- 未选择具体商机时，按时间显示全部商机记忆
- 选择某个商机时，只显示该商机记忆

右侧：

- 统一名称：`关键信息面板`
- 未选择具体商机时，范围为全部商机
- 选择某个商机时，范围为当前商机
- 显示关键人、相关项目与资料

### 3.3 资源加速

用于体现“销售输入信息以换取组织资源”。

功能包括：

- 申请售前支持
- 申请方案专家
- 申请交付评估
- 申请高层陪访
- 申请价格特批
- 申请合同加急
- 自动生成申请材料
- 展示资源 SLA 跟踪

### 3.4 管理驾驶舱

用于管理者查看团队销售预测和风险。

核心模块：

- AI 经营摘要
- 预测金额
- 平均赢率
- 高风险商机数
- 资源请求数
- 团队商机预测表
- 需要管理层介入的商机

## 4. AI 输入体验

### 4.1 输入方式

销售在输入框中输入自然语言，例如：

```text
客户确认后续要走招投标，采购会发标书模板，需要我们准备技术标、商务标和投标文件。
```

系统不要求销售先选择固定阶段，也不要求填写大量字段。

### 4.2 实时关键词高亮

输入时，上方实时高亮区会识别并高亮以下类型：

| 类型 | 示例 | 颜色/样式建议 |
|---|---|---|
| 预算/金额 | 80万、120万 | 蓝色标签 |
| 时间 | 8月、下周、月底 | 绿色标签 |
| 竞品/竞争 | A厂商、竞品、竞争对手 | 红色标签 |
| 决策链 | CIO、老板、最终拍板人 | 紫色标签 |
| 风险/约束 | 压价、定制、上线、付款、验收 | 橙色标签 |
| 系统关键词 | CRM、CPQ、ROI、ERP、LTC | 青色标签 |

这些高亮词必须可以点击。

点击高亮词后，弹出信号解释卡，内容包括：

- 信号类型说明
- 对当前商机的含义
- 建议动作

例如点击 `A厂商`：

- 说明：这是竞争态势信号
- 含义：竞争风险会上升
- 建议：准备竞品对比、差异化价值材料、ROI 证明和价格谈判策略

### 4.3 AI 理解卡

点击 `生成 AI 理解卡` 后，系统不立即更新商机，而是先生成一个确认卡。

AI 理解卡包含：

- 系统诊断摘要
- 识别到的业务信号
- 如果有推进路径变化，显示路径变动提示

销售确认无误后，点击：

```text
确认更新商机
```

此时系统才真正更新：

- 商机状态
- 赢率
- 风险
- 权益等级
- 动态推进路径
- 任务
- 资源建议
- 商机记忆

## 5. 规则引擎设计

当前 Demo 使用前端规则模拟 AI。

### 5.1 信号识别

从文本中识别：

- 金额：`(\d+)(万|万元)`
- 时间：`8月`、`9月底`、`下周`、`本月`、`季度`
- 竞品：`A厂商`、`B厂商`、`竞品`、`竞争对手`
- 商务风险：`压价`、`降价`、`折扣`、`报价`
- 交付风险：`定制`、`上线`、`工期`、`验收`
- 回款风险：`付款`、`预付款`、`尾款`、`回款`
- 决策链：`CIO`、`CEO`、`老板`、`最终拍板人`
- 关键词：`CRM`、`CPQ`、`ROI`、`ERP`、`LTC`

### 5.2 状态评分维度

每个商机维护以下状态分数：

- 需求清晰度
- 痛点强度
- 预算确定性
- 决策链清晰度
- 关键人支持度
- 采购时间明确度
- 竞争风险
- 方案匹配度
- 商务风险
- 交付风险
- 回款风险

### 5.3 赢率与风险

赢率由正向成熟度和风险综合计算：

正向因素：

- 需求清晰
- 预算明确
- 决策链清楚
- 关键人支持
- 方案匹配
- 采购时间明确

负向因素：

- 竞争风险
- 商务风险
- 交付风险
- 回款风险

风险等级：

- 高
- 中
- 低

### 5.4 权益等级

权益等级为 `L1-L6`：

| 等级 | 含义 | 可获得权益 |
|---|---|---|
| L1 | 基础有效 | 进入销售漏斗 |
| L2 | 需求验证 | 可申请售前初步支持 |
| L3 | 预算明确 | 可申请方案专家/案例 |
| L4 | 决策清晰 | 可申请主管或高层陪访 |
| L5 | 商务成熟 | 可进入快速审批判断 |
| L6 | 签约冲刺 | 跨部门加急支持 |

注意：权益等级不等于固定流程阶段。  
真正的流程展示应使用“动态推进路径”。

## 6. 动态推进路径

这是本系统区别于传统 LTC 的关键功能。

系统不使用固定阶段模板，而是根据输入内容动态插入或移除阶段。

默认路径：

```text
需求澄清 → 方案验证 → 商务报价 → 合同签约 → 交付启动 → 验收回款
```

### 6.1 招投标阶段

如果输入中出现：

```text
招标、投标、招投标、标书、投标文件、开标、中标
```

则路径中插入：

```text
招投标
```

并在商机记忆中记录：

```text
推进路径变动：新增“招投标”阶段，后续需要准备标书、投标策略和评标风险。
```

如果后续输入中出现：

```text
不招标、不走招标、不在招标、不需要招标、不用招标、取消招标、不走招投标、无需招投标
```

则移除：

```text
招投标
```

并在商机记忆中记录：

```text
推进路径变动：移除“招投标”阶段，原因是客户沟通中明确不再走招标。
```

### 6.2 POC/试点验证阶段

如果输入中出现：

```text
POC、试点、演示、样板、验证环境、概念验证
```

则路径中插入：

```text
POC/试点验证
```

如果后续输入中出现：

```text
不做POC、不做试点、取消试点、不需要演示、不用演示
```

则移除该阶段。

## 7. 商机记忆库

### 7.1 数据结构

每次确认 AI 理解卡后，写入一条记忆：

```js
{
  id,
  time,
  text,
  signals,
  changes,
  before,
  after
}
```

其中 `changes` 包括：

- 赢率变化
- 权益等级变化
- 风险等级变化
- 信息完整度变化
- 动态推进路径变化

### 7.2 记忆库展示

商机记忆库为三栏：

左栏：

- 全部商机
- 单个商机列表
- 每个商机显示记忆数量、风险、赢率

中栏：

- 如果选择全部商机：按时间展示全部商机记忆
- 如果选择具体商机：展示该商机记忆

右栏：

- 关键信息面板
- 随左栏选择切换范围

## 8. 关键信息面板

关键信息面板不是单纯关键人列表，而是一个关联信息入口。

### 8.1 关键人展示

显示：

- 姓名
- 角色
- 态度
- 所属商机，只有全局视角显示

角色示例：

- 采购
- 财务
- 业务/技术
- 最终决策
- 关键人

态度示例：

- 支持
- 谨慎
- 未知

### 8.2 相关项目与资料

系统根据当前选择范围召回相似项目。

匹配因素：

- 是否涉及相同关键词：`CRM`、`ERP`、`CPQ`
- 是否都有招投标
- 是否都有 POC/试点
- 预算金额是否接近
- 行业是否相近

相关项目卡片显示：

- 项目名称
- 匹配原因
- 行业
- 金额
- 匹配分

点击相关项目后弹出详情：

- 当时项目的招投标/推进记忆
- 关键经验
- 模拟附件

附件可以是模拟按钮，不需要真实文件。

示例附件：

- 技术标模板.docx
- 商务报价拆分表.xlsx
- 竞品评分对比.pdf
- POC计划书.docx
- ERP接口范围说明.xlsx

## 12. 视觉与交互风格

整体风格：

- 专业 SaaS
- AI cockpit 感
- 不是营销落地页
- 不是传统表格 CRM

PC：

- 左侧深色导航
- 主工作台为上方 AI 输入区 + 下方左右两块
- 卡片边框清晰
- 状态标签语义化颜色

移动端：

- 单栏布局
- 底部固定导航
- 大按钮，适合触控
- 横向滚动商机卡片

- 支持 iOS/Android/HarmonyOS 浏览器

颜色建议：

- 主色：蓝色
- AI 区：深蓝/深色渐变
- 成功/支持：绿色
- 风险/压价：橙色或红色
- 决策链：紫色
- 系统关键词：青色

## 13. PWA 与多端方向

应包含基础 PWA 配置：

```html
<meta name="theme-color" content="#0f172a" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />
<meta name="apple-mobile-web-app-title" content="LTC 2.0" />
<link rel="manifest" href="manifest.json" />
```

`manifest.json` 示例：

```json
{
  "name": "LTC 2.0 AI 销售作战台",
  "short_name": "LTC 2.0",
  "start_url": "./index.html",
  "display": "standalone",
  "background_color": "#eef2f7",
  "theme_color": "#0f172a",
  "orientation": "portrait-primary",
  "icons": []
}
```

多端策略：

1. 第一阶段：响应式 Web/PWA
2. 第二阶段：后端、数据库、登录、多用户
3. 第三阶段：接入真实大模型和知识库
4. 第四阶段：可考虑用原生壳或跨端框架封装 App
