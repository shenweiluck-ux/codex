# 怎么看这个 Demo 成果

这份说明写给没有技术背景的同学，用来快速看到当前已经做出来的 LTC 2.0 智能销售作战 Demo。

## 现在是否已经开发完？

已经完成的是 **第一版可演示 Demo**，不是最终生产系统。

你可以把它理解为一个可点击、可输入、能自动给出结果的原型页面。它已经能演示核心业务流程：

1. 销售录入客户和商机信息。
2. 系统计算商机健康度分数。
3. 系统提示风险预警。
4. 系统生成下一步作战任务。
5. 系统给出资源加速建议。
6. 管理者可以看到管道金额、平均赢率、高风险商机和资源诉求统计。

还没有做的是正式系统常见能力，例如账号登录、数据库、多人协作、CRM 对接和真实 AI 接口。

## 最简单的查看方式

如果你已经在电脑上打开了这个项目文件夹，可以按下面做：

### 方法一：直接打开网页文件

1. 打开项目文件夹。
2. 进入 `frontend` 文件夹。
3. 双击 `index.html`。
4. 浏览器会打开 Demo 页面。

这是最适合非技术用户的方式。

### 方法二：用本地服务打开

如果你会使用终端，可以在项目根目录运行：

```bash
npm start
```

然后在浏览器打开：

```text
http://localhost:3000
```

## 打开后怎么看

进入页面后，你会看到几个区域：

- **销售实时输入**：填写客户名称、商机阶段、金额、赢率、关键情况和下一步计划。
- **商机健康度**：提交后会显示一个分数，用来判断这个商机是否健康。
- **风险预警**：系统会根据你输入的关键词提示风险。
- **动态作战任务**：系统会给销售下一步建议。
- **资源加速方案**：系统会生成需要申请哪些资源的摘要。
- **管理者驾驶舱**：展示整体管道金额、平均赢率、高风险商机和资源诉求数量。
- **跟进历史**：保存最近 10 条演示记录，刷新页面后也能看到。

## 可以试着输入的示例

你可以复制下面这个示例来体验：

- 客户名称：华东制造集团
- 商机阶段：方案验证
- 预计金额：260
- 当前赢率：45
- 关键情况：客户关注预算和交付风险，竞品已经进入，当前需要技术专家支持方案验证。
- 下一步计划：本周五完成技术澄清并提交 ROI 测算。

点击 **生成作战建议** 后，你应该会看到：

- 健康度分数发生变化。
- 出现预算、竞争、交付相关风险。
- 出现方案验证阶段的下一步任务。
- 资源加速方案里出现专家、交付或商务报价等资源建议。

## 如果你要给别人演示

建议演示顺序：

1. 先说明这是第一版 Demo，用来验证业务流程，不是最终系统。
2. 填一条示例商机。
3. 点击“生成作战建议”。
4. 从上到下讲解评分、风险、任务、资源方案和驾驶舱。
5. 说明后续可以继续升级为带登录、数据库、AI 分析和 CRM 对接的正式系统。
